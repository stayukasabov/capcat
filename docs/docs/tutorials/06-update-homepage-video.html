Update the Homepage Video

The homepage features an embedded YouTube player. This tutorial explains how to swap the video, and how to push the change to the live website at capcat.org.

Prerequisites

  • Git installed and configured
  • Access to the capcat GitHub repository
  • A text editor (VS Code, Zed, nano — any will do)

Step 1 — Find the YouTube video ID

A YouTube video URL looks like this: 

https://www.youtube.com/watch?v=dQw4w9WgXcQ

The video ID is the value after v=. In this example: dQw4w9WgXcQ. Copy the ID of the video you want to use.

Step 2 — Open the homepage file

The homepage is at: 

docs/index.html

Open it in your editor.

Step 3 — Replace the video ID

Search the file for data-plyr-embed-id. You will find this block: 

<div class="hero-demo__player"
     data-plyr-provider="youtube"
     data-plyr-embed-id="dQw4w9WgXcQ">
</div>

Change the value of data-plyr-embed-id to the new video ID. For example: 

data-plyr-embed-id="YOUR_VIDEO_ID_HERE"

Save the file.

The player is set to noCookie: true and rel: 0. This means YouTube won't use tracking cookies and won't show unrelated videos after playback. These settings are in docs/js/main.js and don't need to change when you swap the video.

Step 4 — Verify locally

Open docs/index.html directly in a browser (no server needed) and confirm the correct video appears in the player. The Plyr controls should be orange.

Step 5 — Commit the change

The website is hosted on GitHub Pages from the main branch. Every push to main automatically deploys to capcat.org within about 60 seconds.

Stage and commit the file:

git add docs/index.html
git commit -m "content: update homepage video"

Step 6 — Push to production

git push origin main

That's the entire deployment. GitHub Pages detects the push and rebuilds the site. Open capcat.org in a browser after ~60 seconds to confirm the new video is live.

If the old video still appears, do a hard refresh: Cmd+Shift+R on Mac or Ctrl+Shift+R on Windows/Linux. GitHub Pages may also cache for a few minutes — wait and try again if needed.

How the player works

The player is Plyr 3.7.8, loaded from CDN. It wraps the YouTube IFrame API with custom controls skinned to the brand orange (#f1540d). The initialization is in docs/js/main.js: 

new Plyr('[data-plyr-provider="youtube"]', {
    controls: ['play-large', 'play', 'progress',
               'current-time', 'mute', 'volume', 'fullscreen'],
    youtube: { noCookie: true, rel: 0,
               showinfo: 0, modestbranding: 1 }
});

To change the player accent colour, update --plyr-color-main in docs/css/main.css. It is currently bound to var(--accent-primary) from the design system.
