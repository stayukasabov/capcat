User Config YAML — Property Inventory


Date: 2026-04-02 Status: Brainstorming / Scoping Context: Design of a user-editable YAML config with documented fields for post-install tuning, vault-centric with user-level defaults fallback.

Config Hierarchy (existing)


Config/capcat.yml              — Vault-level (primary, per-vault)
~/.config/capcat/config.yml   — User-level (machine-wide defaults)
/etc/capcat/config.yml        — System-level (rarely used)

Vault config overrides user config. User config overrides system defaults.

Group 4 — New fields (require config support + enforcement code)


These do not exist anywhere in the codebase yet. Both config entry and enforcement logic need to be written.


| Field | Suggested default | Description |

|---|---|---|
