Breadcrumb Phase 2: HTML Rendering Standardisation


Date: 2026-03-15 Scope: Enforce consistent breadcrumb display in _generate_breadcrumb and add 3rd-level breadcrumb for comments pages.

Prerequisite


Phase 1 complete. After Phase 1, _build_breadcrumb_path returns folder names with underscores converted to spaces. generate_article_html_from_template receives:


  • Article page: breadcrumb_path = [date_folder, source_folder] — 2 elements
  • Comments page: breadcrumb_path = [date_folder, source_folder, "Comments"] — 3 elements (post_processor always appends "Comments")

After Phase 1 the values look like:

  • breadcrumb_path[0] = "News 15-03-2026"
  • breadcrumb_path[1] = "Hacker-News 15-03-2026"
  • breadcrumb_path[2] = "Comments" (comments pages only)


Desired Breadcrumb Output


Article page: 
News 15 March 2026 > Hacker News

Comments page: 
News 15 March 2026 > Hacker News > Article Title

The CSS triangle separator between <a> tags is already defined — not rendered in HTML.



Changes


1. capcat/core/html_generator.py — generate_article_html_from_template


Move is_comments_page determination to before the _generate_breadcrumb call (currently it's after). The code block at lines 567–571 must be repositioned above the breadcrumb call at lines 560–564.

Then, when is_comments_page is True, build a local 3-element list for rendering:


base_article_title = re.sub(r'\s*-\s*Comments$', '', article_title, flags=re.IGNORECASE).strip()
breadcrumb_for_render = list(breadcrumb_path[:2]) + [base_article_title]

Pass breadcrumb_for_render to _generate_breadcrumb instead of breadcrumb_path.

When is_comments_page is False, pass breadcrumb_path directly (no change).


Critical: breadcrumb_path itself is never mutated. The 3rd element is base_article_title (stripped of - Comments), replacing the "Comments" string that was at index 2. The resulting list is always 3 elements for comments pages.

The "Comments" in breadcrumb_path check in is_comments_page remains — it still works because the post_processor continues to pass "Comments" as breadcrumb_path[2].


2. capcat/core/html_generator.py — _generate_breadcrumb


Depth table (unchanged): 
news.html            → base_depth=1  (date only)
article.html         → base_depth=2  (date + source)
html/article.html    → base_depth=3  (date + source)
html/comments.html   → base_depth=3  (date + source + article)

filenames list: change from ["index.html", "news.html"] to ["index.html", "news.html", "article.html"] where index 2 is explicitly "article.html" — the comments crumb links back to the article, not to comments itself.
show_items selection: add comments branch: 
if is_source_index:
    show_items = breadcrumb_path[:1]
    base_depth = 1
elif html_subfolder and current_file_path == "comments.html" and len(breadcrumb_path) >= 3:
    show_items = breadcrumb_path[:3]
    base_depth = 3
elif html_subfolder:
    show_items = breadcrumb_path[:2]
    base_depth = 3
else:
    show_items = breadcrumb_path[:2]
    base_depth = 2

Level formatting — delete line 866 (cleaned_title = self._clean_title_for_display(item)) and the if i >= 1 block below it (lines 867–869). Replace them entirely with a three-branch if/elif/else per-level logic below. The _clean_title_for_display method itself is not deleted — it is still called from lines 670 and 709 elsewhere in the file:
  • i == 0 (date folder): extract DD-MM-YYYY via regex \d{2}-\d{2}-\d{4}, reformat as "News {int(day)} {month_name} {year}". Fallback if regex fails: use item as-is.
  • i == 1 (source folder): strip trailing \s+\d{2}-\d{2}-\d{4}, then replace all - with .
  • i == 2 (article title, comments only): use item as-is — already clean string from the caller.

href arithmetic for Level 2: i=2, base_depth=3, levels_up = 3-2 = 1, href = "../" + "article.html" = "../article.html" ✓
