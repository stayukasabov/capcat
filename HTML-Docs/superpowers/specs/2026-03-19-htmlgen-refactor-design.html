HTML Generation Refactor — Design Spec


Date: 2026-03-19 Status: Approved Approach: TDD Red → Green → Refactor, no parallel path, delete old file at end

Problem


capcat/core/html_generator.py is 1,836 lines. It is a single class with private helpers,

string-matched template logic, and navigation assembly all mixed together. Every change risks

unintended side-effects. The file must be read in full to understand any single feature.



Scope


Replace HTMLGenerator with a clean capcat/htmlgen/ module. Output HTML must be

byte-for-byte identical to the current output. No behaviour changes in this refactor.

Out of scope: source-specific subclasses (HnGenerator etc.). No source currently requires

custom HTML generation logic — all sources share the same template path. The factory

extensibility point is included for future use but no subclasses are written now.

Two call sites: html_post_processor.py and article_fetcher.py.

Target Architecture


capcat/htmlgen/
    __init__.py          → exports ArticleHTMLGenerator, HTMLGeneratorFactory
    generator.py         → ArticleHTMLGenerator (replaces HTMLGenerator)
    factory.py           → HTMLGeneratorFactory (registry, extensibility point)

ArticleHTMLGenerator


Single class. Owns:

  • generate_article_html_from_template() — full pipeline: read md → clean → convert →

transform → breadcrumb → navigation → render via TemplateRenderer

  • generate_directory_index() — determine root vs source level, render via TemplateRenderer
  • _clean_title_for_display() — strip date suffix, normalise hyphens. Underscore prefix

preserved intentionally: call site uses self.html_generator._clean_title_for_display()

and the convention is maintained in the new class.


All private helpers migrated verbatim from html_generator.py:

  • _generate_breadcrumb(), _generate_directory_listing(), _generate_index_navigation()
  • _generate_comments_navigation(), _create_navigation_container()
  • _count_comments(), _clean_markdown_content(), _remove_grey_placeholder_images()
  • _remove_image_anchor_wrappers(), _remove_headerlink_anchors()
  • _remove_duplicate_h1_title(), _wrap_source_url_in_div()
  • _adjust_paths_for_subfolder(), _get_article_title_from_markdown()
  • _generate_error_page()

Dependencies: TemplateRenderer, DesignSystemCompiler (via TemplateRenderer), markdown

processor, find_comments_md from storage_manager.


HTMLGeneratorFactory


class HTMLGeneratorFactory:
    @staticmethod
    def create() -> ArticleHTMLGenerator:
        return ArticleHTMLGenerator()

Minimal for now. Registry pattern (subclass registration) added when first source-specific

generator is needed.



TDD Plan


Phase 1 — Red (write failing tests)


Location: tests/unit/test_htmlgen/


test_generator_article.py

All tests use tmp_path (pytest fixture) to create real markdown files on disk.


  • test_generate_article_html_returns_html_string — result is str, contains <html, <body
  • test_generate_article_html_embeds_title — <title> tag contains article_title
  • test_generate_article_html_comments_page_has_back_to_article_nav

— markdown_path contains "comments"; assert result contains class="index-link" and

href="article.html". Tests structural navigation marker, not template wording.

  • test_generate_article_html_single_article_has_no_navigation

— is_single_article=True; assert "navigation-container" not in result

  • test_generate_article_html_with_comments_md_shows_comments_button

— Fixture: tmp_path/article.md + tmp_path/article-Comments.md. Assert

"comments-link" in result.

  • test_generate_article_html_without_comments_md_hides_comments_button

— Fixture: tmp_path/article.md only (no Comments file). Assert

"comments-link" not in result.


Fixture layout for comments-related tests:

tmp_path/
    My-Title.md          # article markdown (markdown_path points here)
    My-Title-Comments.md # present or absent depending on test
find_comments_md searches the parent directory of markdown_path — pointing markdown_path

at tmp_path/My-Title.md means it searches tmp_path/. No mocking needed.


test_generator_directory.py
  • test_generate_directory_index_returns_html_string

— result is str, contains <html

  • test_generate_directory_index_root_level_title_in_result

— directory named News_19-03-2026, title "News Archive";

assert title string appears in result. Both templates are identical; distinction is in

generated content, not template CSS class. Title injection verifies the root path ran.

  • test_generate_directory_index_source_level_title_in_result

— directory named Hacker-News_19-03-2026, title "Hacker News";

assert title string appears in result.

  • test_clean_title_for_display_strips_date_suffix

  • test_clean_title_for_display_normalises_hyphens

— _clean_title_for_display("Hacker-News_19-03-2026") → contains "Hacker News"

  • test_factory_create_returns_new_instance_each_call

Error Handling


Generator raises no exceptions to callers. All errors return an error-page HTML string

via _generate_error_page(), matching current behaviour.
