PRD: Obsidian Frontmatter for Article Output


Status: Draft Date: 2026-03-31 Version target: 1.7.0 (minor — new behaviour, backwards compatible)

Problem


Capcat already produces clean Markdown with bidirectional wikilinks between articles and comments (→ [[comments|Comments]] / ← [[article|Article]]). The output lands in Obsidian and is readable.


But the notes are not queryable. A professional with 500+ archived articles has no way to answer:


  • "Show me all HN articles from this week"
  • "List everything in the tech bundle I captured in March"
  • "Which BBC articles reference climate?"

Without YAML frontmatter, Obsidian's Properties panel is empty, Dataview queries return nothing, and the graph view has no metadata to filter on. The notes exist but are inert.



Target User


Developer/power user professional who:

  • Runs Capcat on a schedule (cron or manual) to archive curated sources
  • Uses Obsidian as a knowledge base (graph view, Dataview plugin, tags)
  • Shares output folders — structured data directories, not links
  • Has an existing vault where Capcat output lands as a subfolder


Goal


Every article.md Capcat produces must begin with YAML frontmatter that makes the note:


  1. Queryable via Dataview
  2. Filterable by source, category, and date in Obsidian graph view
  3. Identifiable by origin URL for deduplication and reference


What Data Is Available


Confirmed available at write time (no new fetching required):


| Field | Source in code |

|---|---|

| title | always present |

| url | article URL, always present |

| date | FeedItem.published_date (Optional) |

| source | source_name from YAML config (e.g. "Hacker News") |

| source_code | source_code passed to ArticleFetcher (e.g. "hn") |

| category | category from YAML config (e.g. "tech", "ai") |

| captured | datetime.now() at write time — always available |


Not available without new work: author (not in FeedItem). Out of scope for v1.7.0.

Frontmatter Spec


article.md


---
title: "Article Title Here"
url: "https://example.com/article"
source: "Hacker News"
source_code: "hn"
category: "tech"
date: 2026-03-28
captured: 2026-03-31
tags:
  - hn
  - tech
---

Rules:
  • title — sanitized (no quotes that break YAML). Wrap in double quotes always.
  • url — original article URL, not the feed URL.
  • source — human-readable name from YAML config (name: field).
  • source_code — machine code from registry (e.g. hn, lb, bbc).
  • category — from YAML config (category: field). Fallback: "unknown".
  • date — FeedItem.published_date formatted as YYYY-MM-DD. If None: omit the field entirely (do not write date: null).
  • captured — datetime.now() at write time, always YYYY-MM-DD.
  • tags — list containing source_code + category. Always at least one entry.

comments.md


Simpler — comments are derivative of the article:


---
title: "Comments: Article Title Here"
article_url: "https://example.com/article"
source_code: "hn"
captured: 2026-03-31
tags:
  - comments
  - hn
---


Dataview Queries This Unlocks


TABLE date, source FROM #tech WHERE captured >= date(today) - dur(7 days)
SORT date DESC


LIST FROM #hn WHERE date >= date(2026-03-01)


TABLE source, url FROM "" WHERE file.name CONTAINS "AI" AND #tech


Behaviour


  • Default on. No flag required. All output gets frontmatter.
  • Idempotent. If a file already has frontmatter on line 1 (---), do not rewrite it. (Consistent with existing wikilink idempotency pattern in storage_manager.inject_comments_wikilink.)
  • HTML output unaffected. The HTML post-processor strips frontmatter before rendering. No change needed there.
  • Single-URL mode included. capcat single <url> also gets frontmatter. source_code falls back to "single", category to "single".


Out of Scope (v1.7.0)


  • author field — not in FeedItem, would require per-source scraping changes
  • cssclasses — Obsidian theme customisation per source; future nice-to-have
  • Retroactive frontmatter injection into existing archives — separate utility if ever needed
  • Frontmatter for PDF/audio/video placeholder notes — defer


Implementation Surface


Three files need changes:


| File | Change |

|---|---|

| capcat/core/storage_manager.py | New inject_frontmatter(path, metadata) function |

| capcat/core/unified_source_processor.py | Call inject_frontmatter after article write |

| capcat/core/streamlined_comment_processor.py | Call inject_frontmatter after comments write |


FeedItem does not need changes — published_date is already there. ArticleFetcher already receives source_code — no interface changes.

Success Criteria


  • Every article.md produced by capcat bundle, capcat fetch, and capcat single begins with valid YAML frontmatter
  • Frontmatter is parseable by python-frontmatter and Obsidian without errors
  • Existing wikilinks (→ [[comments|Comments]]) appear after the closing ---, not inside it
  • Dataview query TABLE source, date FROM #tech returns results from a fresh run
  • All existing tests pass
  • New regression tests cover: frontmatter present, correct fields, idempotency, None date omitted
