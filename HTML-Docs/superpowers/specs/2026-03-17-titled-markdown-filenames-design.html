Design Spec: Titled Markdown Filenames


Date: 2026-03-17 Status: Approved

Problem


Capcat writes all article content to article.md and all comments to comments.md, regardless of the article title. The actual title exists only in the parent folder name (truncated for filesystem safety). In Obsidian graph view, every article node appears as article — making the graph useless for browsing a large archive.


Goal


Use the article title as the markdown filename so Obsidian graph view shows real article titles. HTML generation must not be affected.


Naming Convention


Article markdown: 
What-makes-Intel-Optane-stand-out.md

Comments markdown: 
What-makes-Intel-Optane-stand-out-Comments.md

  • Same sanitization as the folder (sanitize_filename() from capcat/core/utils.py)
  • Character limit: 200 chars for the base stem (title without extension). The -Comments suffix is appended after truncation, so the full comments filename may be up to 209 chars — still within all OS per-filename limits (255 bytes).
  • Folder stays capped at 100 chars for Windows MAX_PATH safety (intermediate path segment)
  • Markdown filename is the leaf node — has ~85 chars of Windows headroom above the folder
  • -Comments is a reserved suffix; no article title may end in -Comments after sanitization (sanitize_filename strips special chars making collision impossible in practice)
  • HTML output filenames (article.html, comments.html) and all generated links are unchanged

Components


1. capcat/core/storage_manager.py — new helpers


Two write helpers and two discovery helpers. All other components import from here — single source of truth for the naming convention.


def article_md_filename(title: str) -> str:
    """Return sanitized markdown filename for an article (e.g. 'My-Title.md').

    The base stem is truncated at 200 chars before adding the extension.
    """
    return sanitize_filename(title, max_length=200) + ".md"

def comments_md_filename(title: str) -> str:
    """Return sanitized markdown filename for comments (e.g. 'My-Title-Comments.md').

    The base stem is truncated at 200 chars; '-Comments.md' is appended after truncation.
    """
    return sanitize_filename(title, max_length=200) + "-Comments.md"

def find_article_md(folder: Path) -> "Path | None":
    """Return the article markdown path in folder, or None if absent.

    Non-recursive: only searches direct children of folder.
    Returns the first .md file whose stem does not end in '-Comments'.
    """
    return next(
        (p for p in folder.glob("*.md") if not p.stem.endswith("-Comments")),
        None
    )

def find_comments_md(folder: Path) -> "Path | None":
    """Return the comments markdown path in folder, or None if absent."""
    return next(folder.glob("*-Comments.md"), None)

Error handling: find_article_md and find_comments_md return None on empty folders or missing files — callers guard with if path is not None before reading. find_article_md uses non-recursive glob so stray .md files in images/ or other subdirectories are not matched.

Data Flow


capcat fetch hn └─ article_fetcher: title = "What makes Intel Optane stand out" └─ storage_manager.create_article_folder(base, title) → folder "What-makes-Intel-Optane-stand-out/" └─ storage_manager.save_article_content(folder, content, title) └─ filename = article_md_filename(title) → "What-makes-Intel-Optane-stand-out.md" └─ hn/source.py saves comments: └─ filename = comments_md_filename(title) → "What-makes-Intel-Optane-stand-out-Comments.md" capcat fetch hn (HTML generation) └─ html_generator._generate_directory_listing(path) └─ find_article_md(item) → "What-makes-Intel-Optane-stand-out.md" (not None → it's an article) └─ find_comments_md(item) → "What-makes-Intel-Optane-stand-out-Comments.md" (not None → has comments) └─ href = "item.name/html/article.html" ← unchanged

Error Handling


  • find_article_md / find_comments_md return None — all callers must null-check before reading
  • If a folder contains multiple .md files not ending in -Comments (possible from manual edits): find_article_md returns the first match alphabetically; no error raised
  • save_article_content gains a required title parameter — any call site that cannot provide a title is a bug and must be fixed before merging

Migration


Forward-only. Existing article.md / comments.md files on disk are not renamed. After the change, find_article_md will not match the old filenames — those folders will appear to have no article to the generator.
Re-fetching any source regenerates both the markdown files (with new names) and the HTML. Users who want titled filenames on old content simply re-fetch.
No mixed-state ambiguity: the discovery helpers match only the new convention. Old folders silently produce no output until re-fetched. This is documented behaviour, not silent corruption.

Testing (tests/unit/test_titled_markdown_filenames.py)


  • article_md_filename("My Title") → "My-Title.md"
  • article_md_filename("A" 250) → stem truncated at 200 chars, result ends in .md
  • comments_md_filename("My Title") → "My-Title-Comments.md"
  • comments_md_filename("A" 250) → stem truncated at 200 chars, result ends in -Comments.md
  • find_article_md(folder) returns path when titled .md exists
  • find_article_md(folder) returns None when folder is empty
  • find_article_md(folder) ignores -Comments.md files
  • find_comments_md(folder) returns path when *-Comments.md exists
  • find_comments_md(folder) returns None when absent
  • storage_manager.save_article_content(folder, content, title) writes to titled filename
  • HTML generator discovers article folder via find_article_md (listing shows article)
  • HTML generator counts comments via find_comments_md (comment count correct)
  • HTML generator selects article-with-comments template when find_comments_md returns a path
  • Post-processor _is_article_directory returns True when titled .md exists

Constraints


  • HTML output filenames (article.html, comments.html) and all generated links unchanged
  • sanitize_filename() unchanged — reused with max_length=200 for the stem
  • No migration of existing files — forward-only; old article.md folders are invisible to generator until re-fetched
  • -Comments is a reserved suffix appended after stem truncation
  • find_article_md / find_comments_md are the single source of truth for discovery — no other code constructs these paths directly
