Breadcrumb Phase 1: Folder Naming Standardisation


Date: 2026-03-15 Scope: Fix inconsistent source folder naming so all folders follow Source-Name_DD-MM-YYYY

Problem


get_source_folder_name uses display_name.replace(' ', '_'), producing Hacker_News_DD-MM-YYYY.

The canonical format is Hacker-News_DD-MM-YYYY — hyphens as word separators, underscore before the date only.


This creates two downstream problems:


  1. extract_source_id in html_generator.py mirrors the same replace(' ', '_') logic to do the reverse mapping — so it will fail to identify source folders once they use hyphens.
  2. _is_archive_root in html_post_processor.py has a dead source-folder branch (space suffix instead of underscore), plus a dead sg_ prefix check from a removed source.


Canonical Folder Format


Source-Name_DD-MM-YYYY        ← source folder (inside date folder)
News_DD-MM-YYYY               ← date folder (parent)

Examples: Hacker-News_15-03-2026, BBC-News_15-03-2026, Al-Jazeera_15-03-2026



Changes


1. capcat/core/utils.py — get_source_folder_name


Change replace(' ', '_') → replace(' ', '-') in both code paths:


  • Registry path: config.display_name.replace(' ', '-')
  • Legacy fallback path: display_name.replace(' ', '-')

Result: "Hacker News" → Hacker-News, "BBC News" → BBC-News, "Al Jazeera" → Al-Jazeera


2. capcat/core/html_generator.py — extract_source_id (line 943)


Change config.display_name.replace(' ', '_') → config.display_name.replace(' ', '-')


Must mirror the same logic as get_source_folder_name exactly. Also update the function docstring to reflect the new canonical format (Hacker-News_DD-MM-YYYY, not Hacker_News_DD-MM-YYYY).


3. capcat/core/html_post_processor.py — _is_archive_root


Three changes:


a) Remove "sg_" from startswith(("news_", "sg_")) → becomes startswith("news_")


b) Fix the dead source-folder branch (line 517): change space suffix to underscore suffix:

  • Before: if name.startswith(f"{folder_name.lower()} "):
  • After: if name.startswith(f"{folder_name.lower()}_"):

c) Update the docstring comment to reflect one standard: Source-Name_DD-MM-YYYY only, remove sg_DD-MM-YYYY reference.


4. capcat/core/html_generator.py — sg_ comments (lines ~1020 and ~1062)


Replace the comment # For Capcats, allow all directories (sg_ captures don't have source_id) with # For Capcats, allow all directories — source_id matching does not apply in both locations. Preserves the intent (Capcats bypass orphan filtering) without the dead sg_* reference.



Acceptable Regression


Existing folders on disk created with the old Hacker_News_DD-MM-YYYY format will no longer be recognised by _is_archive_root or matched by extract_source_id. This is intentional — one standard, no backwards compatibility. Old folders must be renamed or regenerated.



Post-Fix _build_breadcrumb_path Output


After this fix, _build_breadcrumb_path on a path inside Hacker-News_15-03-2026 returns:


["News 15-03-2026", "Hacker-News 15-03-2026", "article-dir-name"]

(_build_breadcrumb_path does path.name.replace("_", " ") — hyphens are preserved.)


The breadcrumb renderer receives breadcrumb_path[:2]:

  • [0] = "News 15-03-2026" — date folder, date still in DD-MM-YYYY form with hyphens
  • [1] = "Hacker-News 15-03-2026" — source folder with trailing date and internal hyphens

Phase 2 will reformat these into "News 15 March 2026" and "Hacker News" respectively. Phase 1 does not touch the renderer.
