Global-Settings YAML — Design Spec


Date: 2026-04-02 Status: Draft Scope: Vault-centric user configuration via annotated YAML file

Goal


Expose all tunable capcat settings as a human-editable Global-settings.yaml file in the vault directory, with a user-level fallback at ~/.config/capcat/Global-settings.yaml. Every field is documented with inline comments, default values, and examples. Starts with PDF limit enforcement (highest user value), then surfaces all existing settings.



Architecture


Two files, one per scope level:


Config/
  capcat.yml              — sources, bundles (existing, CLI-managed, unchanged)
  Global-settings.yaml    — all tunable settings (human-edited, new)

~/.config/capcat/
  Global-settings.yaml    — user-level defaults (auto-created by capcat init, new)

Config load order (lowest → highest priority):


dataclass defaults
  ↑
~/.config/capcat/Global-settings.yaml   (user-level, unconditional pass)
  ↑
Config/Global-settings.yaml             (vault-level, unconditional pass)
  ↑
CAPCAT_* environment variables
  ↑
CLI flags

capcat.yml is loaded separately for sources/bundles only and never participates in settings resolution. CLI arguments and flags are preserved exactly as-is — they always win.
Merge semantics: key-level merging, not section replacement. If vault-level sets only pdf.max_pdf_size_bytes, all other user-level values remain active. Each key overrides its counterpart independently.
article_count resolution: ProcessingConfig.article_count (default 30) is the global default. To make this fallback work, SourceConfig.article_count default must change from 30 to None. _resolve_count in capcat/core/unified_source_processor.py then falls back to config.processing.article_count when the per-source value is None.
max_filename_length: the correct authoritative default is 100. constants.py:MAX_FILENAME_LENGTH = 255 is the filesystem hard limit, not the opinionated capcat cap. After migration, 100 is the only default.

File Format


Annotated YAML. Every field present, commented with description, default, and example where non-obvious. Fields are optional — omitting a field uses the dataclass default.

@dataclass
class PdfConfig:
    max_pdf_size_bytes: int = 20_971_520   # 20MB
    max_pdf_per_article: int = 10

New fields on ProcessingConfig:
  • article_count: int = 30
  • max_images: int = 20
  • max_images_media_mode: int = 1000
  • min_image_dimensions: int = 150
  • max_image_size_bytes: int = 5_242_880
  • conversion_timeout: int = 30
  • max_filename_length already present at default 100 — no change

New fields on NetworkConfig:
  • crawl_delay: float = 1.0
  • robots_cache_ttl_minutes: int = 15

FetchNewsConfig: add pdf: PdfConfig = None, initialize in __post_init__.
from_dict() fix: from_dict() currently constructs NetworkConfig, ProcessingConfig, LoggingConfig but silently drops both ui and pdf sections. Fix: add construction of UIConfig and PdfConfig alongside the existing three. All sub-config construction must filter incoming dicts to known fields to prevent TypeError on unknown keys: 
def _filter_fields(cls, data: dict) -> dict:
    known = {f.name for f in dataclasses.fields(cls)}
    return {k: v for k, v in data.items() if k in known}

Apply _filter_fields before every sub-config constructor call in from_dict().


New _load_settings_file(path: Path) method on ConfigManager:

This is a new method (does not replace or modify _merge_config_data):


def _load_settings_file(self, path: Path) -> None:
    if not path.exists():
        return
    data = yaml.safe_load(path.read_text()) or {}
    # sources/bundles are valid in capcat.yml — silently ignore here
    data.pop("sources", None)
    data.pop("bundles", None)
    # Filter each section to known fields before merging
    for section_name, section_data in list(data.items()):
        cls = self._section_to_class.get(section_name)
        if cls and isinstance(section_data, dict):
            data[section_name] = _filter_fields(cls, section_data)
    self._merge_config_data(data)

_section_to_class is a dict mapping section key → dataclass: {"network": NetworkConfig, "processing": ProcessingConfig, "ui": UIConfig, "logging": LoggingConfig, "pdf": PdfConfig}

Called twice in load_config() — user-level path first, vault-level second. Exact placement: after self._config = FetchNewsConfig() (the defaults reset) and before the if config_file: branch. This ensures:

  • Settings are applied on top of fresh defaults, not stale state
  • Global-settings files are loaded regardless of whether --config is used
  • Not re-loaded on subsequent calls (gated by the _config_loaded guard above)

Ordering dependency: FetchNewsConfig must have pdf: PdfConfig = None (initialized in __post_init__) before _load_settings_file is wired — otherwise _merge_config_data will log "Unknown config section: pdf" and drop all pdf values.

capcat/core/ethical_scraping.py


Add self.crawl_delay: float = 1.0 to EthicalScrapingManager.__init__ (after self.last_request_time). This ensures enforce_rate_limit can reference self.crawl_delay safely before configure() is called.


Add a configure(crawl_delay: float, robots_cache_ttl_minutes: int) method to EthicalScrapingManager. This method updates self.crawl_delay and self.cache_ttl on the already-constructed singleton:


def configure(self, crawl_delay: float, robots_cache_ttl_minutes: int) -> None:
    self.crawl_delay = crawl_delay
    self.cache_ttl = timedelta(minutes=robots_cache_ttl_minutes)

__init__ keeps its existing hardcoded defaults (1.0 and 15) — these are the fallback if configure() is never called.
Wiring crawl_delay into enforce_rate_limit: enforce_rate_limit currently has signature (self, domain, crawl_delay, min_delay=1.0). Change the body to use self.crawl_delay as an additional floor: 
effective_delay = max(crawl_delay, min_delay, self.crawl_delay)

This means calls that pass an explicit min_delay still respect it, and the configured self.crawl_delay acts as the global minimum. No call-site changes needed.


article_fetcher.py calls get_ethical_manager().configure(config.network.crawl_delay, config.network.robots_cache_ttl_minutes) once at startup before any fetching begins.

capcat/core/async_pdf_manager.py


Singleton seeding: add initialize_pdf_manager(pdf_config: PdfConfig) that creates and caches the global singleton with the given config. get_pdf_manager() falls back to PdfConfig() defaults if called before initialization.
AsyncPDFManager.__init__ accepts pdf_config: PdfConfig = None. Defaults to PdfConfig() if not provided. Add self._check_session = requests.Session() to __init__ for use in the size pre-flight check.
Count enforcement — extract_and_queue_pdf_links: stop queuing after pdf_config.max_pdf_per_article URLs per article. Log a debug message when the cap is hit.
Size enforcement — insert the following check at the start of _download_pdf, before the download_file(...) call: 
# Pre-flight size check
try:
    head = self._check_session.head(url, timeout=10, allow_redirects=True)
    content_length = head.headers.get("Content-Length")
    if content_length and int(content_length) > self.pdf_config.max_pdf_size_bytes:
        logger.warning("Skipping PDF (too large: %d bytes): %s", int(content_length), url)
        return
except Exception:
    pass  # If HEAD fails, allow the download to proceed

This replaces the existing _check_pdf_size_and_prompt() interactive prompt. The interactive prompt is removed — size gating is now silent and config-driven.


Direct-PDF URL path: article_fetcher.py has a second call site for _check_pdf_size_and_prompt — the direct-PDF URL path (is_direct_pdf_url check in fetch_article_content). This path bypasses AsyncPDFManager entirely. A shared helper is needed for both paths.
Circular import warning: async_pdf_manager already imports from downloader via deferred import to avoid cycles. article_fetcher imports get_pdf_manager from async_pdf_manager. Importing from article_fetcher inside async_pdf_manager would create a cycle. The helper must live in async_pdf_manager.py and be imported by article_fetcher, not the other way.

Define the helper as a module-level function in async_pdf_manager.py:


def pdf_exceeds_size_limit(url: str, session, max_bytes: int) -> bool:
    try:
        head = session.head(url, timeout=10, allow_redirects=True)
        content_length = head.headers.get("Content-Length")
        if content_length and int(content_length) > max_bytes:
            return True
    except Exception:
        pass
    return False

  • AsyncPDFManager._download_pdf calls pdf_exceeds_size_limit(url, self._check_session, self.pdf_config.max_pdf_size_bytes)
  • article_fetcher.py imports pdf_exceeds_size_limit from async_pdf_manager and calls it in the direct-PDF URL path: pdf_exceeds_size_limit(url, self.session, config.pdf.max_pdf_size_bytes)

Both paths use the same helper and the same configured limit. _check_pdf_size_and_prompt is deleted.

capcat/core/source_system/source_registry.py


source_registry.py constructs SourceConfig with explicit article_count fallbacks at two call sites: 
article_count=int(config.get("article_count", 30)),      # line ~173
article_count=int(config_data.get("article_count", 30)), # line ~251

Both bypass the dataclass default entirely. Change both to omit the fallback:


article_count=int(config.get("article_count")) if config.get("article_count") is not None else None,

Or more simply:

_raw = config.get("article_count")
article_count=int(_raw) if _raw is not None else None,

Without this change, None never propagates and config.processing.article_count is never reached in _resolve_count.


capcat/cli.py


New capcat settings command (independent of the existing generate-config command — do not modify generate-config):

Writes Config/Global-settings.yaml in the current vault directory using the full annotated template. If file exists: print warning and exit. Accept --force to overwrite.


Wire into _dispatch():

elif command == "settings":
    _cmd_settings(rest)

Add settings to _print_help() usage listing.


init command — _cmd_init() pseudocode: 
def _cmd_init(reinit, ...):
    init_project(reinit=reinit, ...)

    if not reinit:
        user_settings = Path.home() / ".config" / "capcat" / "Global-settings.yaml"
        if not user_settings.exists():
            user_settings.parent.mkdir(parents=True, exist_ok=True)
            user_settings.write_text(GLOBAL_SETTINGS_TEMPLATE)

reinit=True skips the user-level file write. GLOBAL_SETTINGS_TEMPLATE is the full annotated YAML string defined as a constant in cli.py (same template used by capcat settings).
