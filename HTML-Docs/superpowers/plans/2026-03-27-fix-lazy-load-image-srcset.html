Fix Lazy-Load Image srcset Placeholder Bug


> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.


Goal: Fix image extraction so lazy-loaded images (WordPress/Avada data:image/svg+xml srcset placeholders) resolve to real URLs instead of blank SVG placeholders.
Architecture: Three parse_srcset implementations exist across the codebase — formatter._parse_srcset, media_processor.MediaProcessor.parse_srcset, and image_processor (which imports from formatter). All must skip data: URI entries. _process_images and _extract_image_urls must also add data-srcset as a fallback attribute for sites where both src and srcset may be empty but data-srcset holds the real responsive URLs.
Tech Stack: Python 3.9+, BeautifulSoup4, pytest, unittest.mock

Task 1: Fix _parse_srcset in formatter.py + _process_images data-srcset fallback


Context: _parse_srcset (lines 208-252) is the shared parser called by both _process_images and image_processor._extract_image_urls. When srcset="data:image/svg+xml,...", _parse_srcset returns "data:image/svg+xml" (truthy), which overwrites the real src. Fix: skip any entry whose URL starts with data:. Additionally, _process_images (lines 263-299) must add data-srcset as a fallback for sites where both src and srcset are placeholders but data-srcset holds real responsive URLs.

def _parse_srcset(srcset: str) -> str:
    """Parse srcset attribute and return the highest resolution image URL.

    Skips data: URI entries (lazy-load SVG placeholders used by WordPress/Avada
    and similar CMS platforms). Returns '' if no real URL is found.
    """
    if not srcset:
        return ""

    entries = [entry.strip() for entry in srcset.split(",")]

    best_url = ""
    max_width = 0

    for entry in entries:
        parts = entry.rsplit(" ", 1)
        if len(parts) == 2:
            url, descriptor = parts
            url = url.strip()
            if url.startswith("data:"):
                continue
            descriptor = descriptor.strip()

            if descriptor.endswith("w"):
                try:
                    width = int(descriptor[:-1])
                    if width > max_width:
                        max_width = width
                        best_url = url
                except ValueError:
                    continue
            elif descriptor.endswith("x"):
                try:
                    float(descriptor[:-1])
                    if not best_url or max_width == 0:
                        best_url = url
                except ValueError:
                    continue
        elif len(parts) == 1:
            url = parts[0].strip()
            if url.startswith("data:"):
                continue
            if not best_url:
                best_url = url

    return best_url
