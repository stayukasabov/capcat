Fix: PDF Downloads Ignore User Choice Implementation Plan


> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.


Goal: Make --media (the "Download attached PDFs?" interactive prompt) actually skip PDF downloads when the user says No.
Architecture: The bug lives in two unconditional call sites in ArticleFetcher._download_pdf_links_from_markdown — both ignore self.download_files. Gating those calls on self.download_files and updating the existing regression test (which locked in the wrong behavior) is the full fix. The interactive prompt text is also updated for clarity.
Tech Stack: Python 3, pytest, capcat/core/article_fetcher.py, capcat/core/async_pdf_manager.py

Root Cause Summary


ArticleFetcher._download_pdf_links_from_markdown is called at two sites in article_fetcher.py without checking self.download_files:

| Line | Caller | Note |

|------|--------|------|

| 1068 | _save_article_with_media | general content path |

| 3044 | fetch_article_content (legacy path) | HN / specialized sources |


A regression test (tests/unit/test_pdf_always_downloads.py) explicitly documents this wrong behavior as intentional, which it will need to be updated.


Additionally, the interactive prompt says "Download attached PDFs?" but maps to --media — once the behavior is fixed this phrasing becomes accurate, though the note text needs a small update.

def test_pdf_not_queued_when_download_files_false(self, tmp_path):
    """When download_files=False, PDF links must NOT be queued."""
    fetcher = self._make_fetcher(download_files=False)
    markdown = "[Paper](https://example.com/research.pdf)\n\nSome content."

    result = fetcher._download_pdf_links_from_markdown(markdown, str(tmp_path))

    assert "downloading_research.pdf" not in result, (
        "PDF must NOT be queued when download_files=False"
    )
    # Original link must be preserved unchanged
    assert "https://example.com/research.pdf" in result

def test_fetch_article_content_skips_pdf_when_flag_false(self, tmp_path):
    """fetch_article_content must not download PDFs when download_files=False."""
    from unittest.mock import patch, MagicMock
    import requests
    from capcat.core.article_fetcher import ArticleFetcher

    class _Fetcher(ArticleFetcher):
        def should_skip_url(self, url, title):
            return False

    session = requests.Session()
    with patch("capcat.core.ethical_scraping.get_ethical_manager") as mock_em:
        mock_em.return_value.configure = MagicMock()
        fetcher = _Fetcher(session, download_files=False)

    queued_pdfs = []

    with patch.object(fetcher, "_download_pdf_links_from_markdown",
                      side_effect=lambda md, path: queued_pdfs.append(path) or md) as mock_dl:
        # Simulate article content with a PDF link
        with patch.object(fetcher, "_fetch_html_content",
                          return_value=(200, "<html><a href='paper.pdf'>Paper</a></html>", "text/html")):
            with patch.object(fetcher, "_save_markdown_file", return_value=None):
                fetcher.fetch_article_content(
                    title="Test", url="https://example.com/article",
                    index=0, base_folder=str(tmp_path)
                )

        mock_dl.assert_not_called(), "PDF downloader must not be called when download_files=False"
