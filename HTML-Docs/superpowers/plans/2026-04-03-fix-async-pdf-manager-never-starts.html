Fix: AsyncPDFManager Worker Never Starts Implementation Plan


> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.


Goal: Fix three bugs that prevent PDF downloads from ever executing during batch article fetches.
Architecture: Three coordinated changes — make initialize_pdf_manager idempotent and auto-start, add a wait_until_idle drain method, and wire shutdown into _process_with_new_system so downloads complete before the batch returns.
Tech Stack: Python threading, concurrent.futures.ThreadPoolExecutor, pytest

Root Cause Summary


B3 — Worker thread never starts initialize_pdf_manager() in capcat/core/async_pdf_manager.py creates an AsyncPDFManager but never calls .start(). Without .start(), no ThreadPoolExecutor and no _worker_loop thread are created. Downloads are queued (queue fills) but nothing drains the queue.
B4 — Manager recreated per article ArticleFetcher.__init__ (line 186) calls initialize_pdf_manager(config.pdf) on every instantiation. HN source creates a new NewsSourceArticleFetcher per article. Each call creates a fresh AsyncPDFManager and overwrites _global_pdf_manager, discarding the previous instance and all queued downloads with it.
B5 — No shutdown/wait after batch _process_articles_with_new_system returns as soon as article futures complete. PDF downloads run in a background thread that outlives the articles loop. Without a drain-and-shutdown call, the program can exit (or move to the next source) while downloads are still pending.

def initialize_pdf_manager(pdf_config: "PdfConfig") -> AsyncPDFManager:
    """Return the running global AsyncPDFManager, creating and starting it if needed.

    Idempotent: subsequent calls with any config return the already-running instance.
    The manager is started on first creation; callers must not call .start() themselves.
    """
    global _global_pdf_manager
    if _global_pdf_manager is not None and _global_pdf_manager.worker_thread is not None:
        return _global_pdf_manager
    _global_pdf_manager = AsyncPDFManager(pdf_config=pdf_config)
    _global_pdf_manager.start()
    return _global_pdf_manager

        manager.start()

    def test_wait_until_idle_waits_for_active_downloads(self, tmp_path):
        """wait_until_idle must block until active_downloads clears."""
        import threading, time

        manager = AsyncPDFManager(max_workers=1)
        manager.start()
        try:
            # Simulate an active download event
            event = threading.Event()
            pdf_url = "https://example.com/slow.pdf"
            with manager._lock:
                manager.active_downloads[pdf_url] = event

            # Complete it after a short delay
            def complete():
                time.sleep(0.2)
                with manager._lock:
                    if pdf_url in manager.active_downloads:
                        manager.active_downloads[pdf_url].set()
                        del manager.active_downloads[pdf_url]

            t = threading.Thread(target=complete)
            t.start()

            result = manager.wait_until_idle(timeout=3.0)
            t.join()

            assert result is True, "Must return True once downloads complete"
        finally:
            manager.stop()

        """wait_until_idle must return False when timeout expires with pending downloads."""
        import threading

        manager = AsyncPDFManager(max_workers=1)
        manager.start()

            # Simulate a download that never completes
            event = threading.Event()

            assert result is False, "Must return False when timeout expires"

def wait_until_idle(self, timeout: float = 60.0) -> bool:
    """
    Block until the download queue is empty and no downloads are active.

    Returns True if idle before timeout, False if timeout expired.
    Used after all articles are processed to drain pending PDF downloads
    before the batch command returns.
    """
    import time as _time
    deadline = _time.monotonic() + timeout
    while _time.monotonic() < deadline:
        with self._lock:
            queue_empty = self.download_queue.empty()
            no_active = len(self.active_downloads) == 0
        if queue_empty and no_active:
            return True
        _time.sleep(0.1)
    return False
