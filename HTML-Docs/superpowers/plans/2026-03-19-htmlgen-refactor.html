HTML Generation Refactor Implementation Plan


> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.


Goal: Replace capcat/core/html_generator.py (1,836-line monolith) with a clean capcat/htmlgen/ module, migrating all call sites and deleting the old file.
Architecture: ArticleHTMLGenerator in capcat/htmlgen/generator.py owns all HTML generation logic (verbatim migration of public methods + private helpers from html_generator.py). HTMLGeneratorFactory in capcat/htmlgen/factory.py is the single construction point. Both call sites (html_post_processor.py and article_fetcher.py) are switched to use the factory. The old file is deleted once all tests pass.
Tech Stack: Python 3, pytest, capcat.core.template_renderer.TemplateRenderer, capcat.core.design_system_compiler.DesignSystemCompiler, markdown library, capcat.core.storage_manager.find_comments_md

"""Factory for creating ArticleHTMLGenerator instances."""
from __future__ import annotations
from capcat.htmlgen.generator import ArticleHTMLGenerator


class HTMLGeneratorFactory:
    """Construction point for HTML generators.

    Minimal now. Registry pattern for source-specific subclasses
    added when the first source requires custom HTML logic.
    """

    @staticmethod
    def create() -> ArticleHTMLGenerator:
        """Return a new ArticleHTMLGenerator instance."""
        return ArticleHTMLGenerator()
