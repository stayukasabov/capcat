## Root Causes / Key Findings

**`capcat.yml` not auto-scaffolded:** `init_project()` in `capcat/commands/init.py` creates `.capcat/`, `Config/`, `Config/themes/`, `Config/sources/active/` -- but never writes `capcat.yml`. That file is optional and user-created. Case study (and diagrams) had documented it as auto-created since the session that wrote the vault architecture section.

**Source count wrong:** The 12 "curated" sources (bundle-ready) were being used as the total count. `capcat list sources` returns 17: 10 config-driven + hn, lb + medium, substack, twitter, vimeo, youtube (custom handlers).

**Stale pipx venv file:** `pipx upgrade` from a local spec path does not remove files deleted from the source. The old `google-reserch.yml` persisted in the venv after the git rename. Required manual deletion.

**pip vs pipx in quick-start.html:** The website was showing `pip install capcat` for macOS/Windows but `pipx install capcat` for Linux only. README and case study had always shown pipx exclusively.
