### v1.6.3 — Lazy-load image srcset fix

**Root cause**: WordPress/Avada lazy-loading sets `srcset="data:image/svg+xml,..."` as a placeholder. `_parse_srcset` split on comma, the SVG data URI contains a literal comma after `data:image/svg+xml,`, producing a dangling fragment that didn't start with `data:` and leaked as a URL. `_is_broken_image_url` also had a gap (checked `"data:image/svg+xml,"` with comma, but extracted result was `"data:image/svg+xml"` without).

**Fixes across three independent implementations:**
- `formatter._parse_srcset`: fragment-merging loop to handle comma-split SVG data URIs; `data:` filter in both branches; `data-srcset` fallback added to `_process_images`
- `image_processor._extract_image_urls`: updated URL-selection priority to mirror formatter (`srcset filtered > src > data-srcset > data-src > data-lazy`)
- `media_processor.parse_srcset`: early return for pure `data:` srcsets; per-entry `data:` filter; rewritten fallback

## Root Causes / Key Findings

**Rate limit 429 warning** (`Firefox & Gtk Emoji picker`): Not a bug. Intentional exponential-backoff retry in `capcat/sources/builtin/custom/lb/source.py:632`.

**Comma-split SVG edge case**: `data:image/svg+xml,%3Csvg...` contains a literal comma after the `,` in `data:image/svg+xml,`. Naive `srcset.split(",")` produces a second fragment (`%3Csvg...`) that doesn't start with `data:` and passed through as a URL.

**`os._exit(130)` mock fallthrough**: Without a `return` after `os._exit(130)` in the mocked test, execution continues to `shutdown_event.set()`, causing the SIGINT test to falsely show event set.

**PyPI migration docs debt**: ~350+ occurrences of stale references (`./capcat`, `../News/`, bare `sources/active/`) accumulated from when capcat was run as a local repo script. None of the bulk-replacement scripts write to `docs/docs/` so they went unfixed until manual audit.
