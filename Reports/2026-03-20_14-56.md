### Userspace Source Config Mirror (v1.4.0)

**10 tasks executed via Subagent-Driven Development, 2 review passes per task (spec compliance + code quality).**

#### Core mirror class
- Created `capcat/core/source_config_mirror.py` — `SourceConfigMirror` class
  - `is_mirrored()`, `run_first_mirror()`, `check_for_upgrades()`
  - Three domains: `config_driven` (`*.yaml`/`.yml`/`.json`), `custom` (full dir with `source.py`), `bundles` (`bundles.yml`)
  - SHA-256 manifest at `.capcat/source_hashes.json`
  - Upgrade diff: Step 1 = new items, Step 2 = hash comparison (3 cases), Step 3 = override with backup
  - Manifest re-sync when file absent but mirror dirs exist
  - TUI mode uses `questionary.confirm`; CLI mode uses `print + input`
  - `questionary is not None` guard prevents crash when package absent

#### Registry / factory singleton bypass
- `get_source_registry(project_root=None)` — bypasses singleton cache when `project_root` is non-`None`
- `get_source_factory(project_root=None)` — same pattern
- 6 new tests in `tests/unit/test_registry.py`

#### UnifiedSourceProcessor wiring
- `__init__` accepts `project_root`; passes it to `get_source_factory`
- `_process_with_new_system` calls `SourceConfigMirror` before article discovery
- `NoProjectError` and broad exceptions caught — mirror failure never blocks a fetch

#### Bundle path routing
- `get_available_bundles(project_root=None)` reads `Config/sources/active/bundles/bundles.yml` when present, falls back to builtin
- `interactive.py` fixed to use `find_project_root()` for bundle path
- `update_manager.py` updated to pass `project_root`
- `cli.py`: three call sites updated (bundle expansion, `_cmd_list` bundles, `_cmd_list` SourceRegistry)
- 3 tests in `tests/unit/test_bundle_routing.py`

#### `add-source` redirect
- `AddSourceService.__init__` accepts `project_root`; redirects `_config_path` and `_bundles_path` to userspace
- `_write_manifest_entry(filename)` — writes `{"builtin_hash": "", "user_hash": "<sha256>"}` to manifest
- `_write_manifest_entry_after_add()` — finds newest file in `_config_path` by mtime and calls above
- `create_add_source_service()` factory resolves `project_root` via `find_project_root()`
- 3 tests in `tests/unit/test_add_remove_source_mirror.py`

- `_remove_manifest_entry(filename)` — removes key from manifest
