## Root Causes / Key Findings

**arXiv PDF links missing:** arXiv puts its PDF download link inside `<aside class="extra-services">`. The cleanup loop calls `soup.find("aside").decompose()` early in `_fetch_web_content`. The old PDF injection code at lines 2967-2999 scanned `soup.find_all("a")` AFTER this decompose — so the link was already gone. Fix: collect all PDF links from the full page before the decompose loop runs.

**Empty folders from PDF processing exception:** `_download_pdf_links_from_markdown` was called after `os.makedirs(article_folder_path)` with no exception guard. An exception in PDF processing propagated out, leaving the folder created but empty. Fix: wrap in try/except, log WARNING, continue.

**Missing PyPI publish:** GitHub Actions `Publish to PyPI` workflow triggers only on `v*` tags, not on pushes to `main`. Pushing the commit without the tag means no publish workflow runs.

**CLAUDE.md lost:** The file is in `.gitignore` (intentional — local-only context). It existed only on disk and was deleted during a previous session's mistake (Claude created a new CLAUDE.md from scratch, user corrected, it was deleted). No git history to restore from.
