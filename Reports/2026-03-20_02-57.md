### Bug Fix: TwitterSource subfolder (v1.3.1)
**Root cause:** `TwitterSource.fetch_article_content` wrote `X.com-post.md` directly into `output_dir` (the HN source directory `Hacker-News_19-03-2026/`) instead of creating a dedicated article subfolder.

**Cascade:**
- `find_article_md(Hacker-News_19-03-2026/)` found the stray file
- `_is_article_directory()` returned True for the whole HN source dir
- HTML post processor generated `html/article.html` for X.com post instead of `news.html` for HN listing
- Root index showed "X.com post" entry; all HN article/comments breadcrumbs had broken `../../news.html` links

**Fix:** Create `output_dir/<sanitized_title>/` subfolder, write inside it, return folder path (matching YouTube/Medium/Vimeo contract).

**Lobsters comments:** Not a code bug — network timeouts fetching `lobste.rs` comment pages. Selectors and logic are correct. Transient.
