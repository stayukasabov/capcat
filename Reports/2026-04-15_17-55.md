### Bugs found during acceptance testing

**BUG F1 — `min_image_dimensions` not enforced for AVIF images**
- Guardian (and BBC) serve AVIF container files with `.jpg`/`.png` extensions.
- `_read_image_dimensions` in `image_processor.py` does not parse AVIF format, returns `None`.
- When `dims is None`, the pixel floor filter is skipped — images pass regardless of configured minimum.
- File: `capcat/core/image_processor.py`, `_download_single_image_filtered()` lines 458–467.

**BUG F2 — `network.connect_timeout` config is dead code**
- `article_fetcher.py` calls `get_timeout_for_source()` which reads from hardcoded `SOURCE_TIMEOUTS` dict in `timeout_config.py`.
- `network.connect_timeout` from `Global-settings.yaml` is never consulted for the main fetch path.
- `session_pool.py` sets `session.timeout` from config but fetcher overrides per-request.
- File: `capcat/core/timeout_config.py`, `capcat/core/article_fetcher.py`.

**BUG F3 — `network.max_retries` config is dead code**
- `network_retry` decorator in `retry.py` hardcodes `max_retries=3`.
- The `config.network.max_retries` path in `exponential_backoff_retry` only executes when `max_retries=None` is passed — which `network_retry` never does.
- File: `capcat/core/retry.py` line 120.
