## Accomplished

### v1.8.4 ‚Äî Three PDF pipeline bugs fixed

**B1 ‚Äî Stale placeholder links (never updated after async download)**
- Root cause: `_schedule_pdf_link_updates` in `article_fetcher.py` used `time.sleep(2)` before calling `update_article_with_completed_downloads`. For articles with many PDFs, downloads take far longer than 2 seconds ‚Äî the update ran too early, found nothing in `completed_downloads`, and exited. Placeholders persisted forever.
- Fix: Added `_queued_by_folder: Dict[str, List[str]]` to `AsyncPDFManager.__init__`. `extract_and_queue_pdf_links` now stores queued URLs per folder under lock. New method `get_queued_urls_for_folder(folder_path)` pops and returns them. The update thread calls `wait_for_downloads(queued_urls, timeout=120)` instead of sleeping.

**B2 ‚Äî Absolute paths written into updated markdown links**
- Root cause: `update_article_with_completed_downloads` used `local_path` directly from `self.completed_downloads[url]`, which stores the absolute path returned by `download_file`.
- Fix: `os.path.relpath(local_path, os.path.dirname(markdown_file_path))` used before constructing the link.

**B3 ‚Äî PDF download bar overflows when articles have many PDFs**
- Root cause: `generator.py` joined PDF links with `" &nbsp;¬∑&nbsp; "` ‚Äî the separator dangled at the start of a new line on wrap. `base.css` `.pdf-download-bar` lacked `flex-wrap: wrap`, and used `line-height: 0.85rem` (wrong ‚Äî same as font-size).
- Fix: CSS block updated with `flex-wrap: wrap`, `align-items: baseline`, `gap: 0.5rem 0.75rem`, `line-height: 1.4`; `.pdf-download-bar a` got `white-space: nowrap`. Generator join changed to `" ".join(`.

## Root Causes / Key Findings

- B1: `time.sleep(2)` is never reliable for async completion. Fix: track queued URLs per folder, wait on actual completion events.
- B2: `download_file` returns an absolute path ‚Äî callers that write to markdown must convert to relative.
- B3: CSS `line-height: 0.85rem` was wrong (no vertical breathing room). `&nbsp;¬∑&nbsp;` separator is inherently unfriendly to line-wrapping ‚Äî plain space + `flex-wrap` is the correct pattern.
- Stray `files/` directory at vault root (`/Volumes/DRIVE B/NewsVault3/files/`) was a 0-byte artifact caused by B1. Actual PDF was correctly saved to the article folder. Safe to delete manually.
