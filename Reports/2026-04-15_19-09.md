## Accomplished

### BUG F3 fixed ‚Äî `network.max_retries` ignored in `network_retry` (v1.9.36)
- **Root cause:** `network_retry` in `retry.py` passed `max_retries=3` explicitly to `exponential_backoff_retry`. The config-reading path inside `exponential_backoff_retry` only executes when `max_retries=None`, so `config.network.max_retries` was always bypassed.
- **Fix:** Changed `max_retries=3` to `max_retries=None` in `network_retry`.
- Regression: `tests/unit/test_retry_config.py` ‚Äî `TestNetworkRetryRespectsConfig` (3 tests).

### BUG F2 fixed ‚Äî `network.connect_timeout` ignored for unknown sources (v1.9.36)
- **Root cause:** `get_timeout_for_source()` in `timeout_config.py` returned `SOURCE_TIMEOUTS["default"]` (hardcoded 10s/30s) for sources not in the dict, never consulting `config.network.connect_timeout` / `read_timeout`.
- **Fix:** Added `from .config import get_config` import; replaced hardcoded default return with a `TimeoutConfig` built from `config.network.connect_timeout` and `config.network.read_timeout`. Per-source `SOURCE_TIMEOUTS` overrides are unchanged and still take precedence.
- Regression: `tests/unit/test_timeout_config.py` ‚Äî `TestGetTimeoutForSourceUsesConfig` (3 tests).

### BUG F1 fixed ‚Äî AVIF images bypass `min_image_dimensions` filter (v1.9.36)
- **Root cause:** `_read_image_dimensions()` in `image_processor.py` parsed PNG, WebP, and JPEG but not AVIF. For AVIF files (ISO Base Media ftyp container), it returned `None`. In `_download_single_image_filtered()`, the pixel floor check is gated on `if dims is not None:`, so AVIF images always passed regardless of configured minimum.
- **Fix:** Added AVIF detection (`header[4:8] == b"ftyp"` + brand check for avif/avis/heic/heif/mif1) followed by an `ispe` box scan (first 4096 bytes) to extract width/height as `(int, int)`.
- Regression: `tests/unit/test_image_processor.py::TestReadImageDimensionsAvif` (3 tests).
