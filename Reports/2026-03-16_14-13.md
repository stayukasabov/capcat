## Root Causes / Key Findings

**Image embedding bug (`capcat/core/image_processor.py`)**

`_process_images()` in `formatter.py` selects the highest-resolution URL from `srcset` when building markdown image references. `_extract_image_urls()` in `image_processor.py` ignored `srcset` entirely, reading only `data-src`/`src`. On WordPress sites `src` is a resized variant (`image-6-1024x768.png`) while `srcset` holds the full-resolution URL (`image-6.png`). The `url_mapping` key was therefore `image-6-1024x768.png` but the markdown contained `image-6.png` ‚Äî string replacement silently failed and all WordPress/responsive images remained as external URLs in the output.

**Fix:** Added srcset-priority logic to `_extract_image_urls` mirroring `_process_images`: `srcset highest-res ‚Üí data-src ‚Üí src`.
